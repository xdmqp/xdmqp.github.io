数据类型

基本类型：

包装类型

与基本类型一 一对应，两者之间的赋值通过自动装箱和拆箱完成。

缓存池

new Integer(123)每次新建一个对象，Integer.valueOf(123)会使用缓冲池中的对象，多次调用会取得同一对象的引用Java8中基本类型对应缓冲池如下：

boolean:true,false

byte:-128~127

short:-128~127

int:-128~127

long:-128~127

char:\u0000~\u007F

使用这些基本类型的包装类型时，如果数值范围在缓冲池范围内，就直接使用缓冲池中的对象

基本数据类型和引用数据类型的区别

基本类型变量名指向具体值，引用数据类型变量名指向存储数据对象的内存地址基本类型变量声明后，jvm会立即给它分配存储空间，引用数据类型变量在声明后不会立即分配内存空间，只存储一个内存地址基本类型使用时需要赋值具体值，使用==判断，引用数据类型使用时可以赋值null，判断一般使用equals方法

String

不可变好处

可以缓存hash值

因为 String 的 hash 值经常被使用，例如 String 用做 HashMap 的 key。不可变的特性可以使得 hash 值也不可变，因此只需要进行一次计算。

String Pool的需要

如果一个 String 对象已经被创建过了，那么就会从 String Pool 中取得引用。只有 String 是不可变的，才可能使用 String Pool。

安全性

String 经常作为参数，String 不可变性可以保证参数不可变。例如在作为网络连接参数的情况下如果 String 是可变的，那么在网络连接过程中，String 被改变，改变 String 的那一方以为现在连接的是其它主机，而实际情况却不一定是。

线程安全

String 不可变性天生具备线程安全，可以在多个线程中安全地使用。

StringBuffer和StringBuilder

StringBuffer和StringBuilder都是可变的其中StringBuffer线程安全，内部使用synchronized同步，StringBuilder不是线程安全的

使用

  1. 操作少量的数据: 适用 String
  2. 单线程操作字符串缓冲区下操作大量数据: 适用 StringBuilder
  3. 多线程操作字符串缓冲区下操作大量数据: 适用 StringBuffer

String Pool

字符串常量池保存着所有字符串字面量(String="xxx")，在编译时期确定。另外，String类型还可以调用intern方法在运行过程中将字符串添加到String Pool中。当调用intern方法时，如果常量池中已经存在与当前字符串值相等的字符串，则返回常量池中字符串的引用；否则，就在常量池中添加一个字符串，并返回这个字符串的引用。

特性

引用类型，final，不可被继承，不可变，java8中用char数组实现，java9中用byte数组加coder标识实现

运算

参数传递

形式：值传递

Java 程序设计语言总是采用按值调用。也就是说，方法得到的是所有参数值的一个拷贝，也就是说，方法不能修改传递给它的任何参数变量的内容。

float与double

隐式类型转换

java不能隐式执行向下转型

关键字

final

声明数据为常量，对于基本类型，使数值不变，对于引用类型，使引用不变；

声明方法不能被子类重写，private方法被隐式指定为final；

声明类不允许被继承

static

  1. 静态变量（类变量）
  2. 静态方法：类方法，不依赖于任何实例，必须要有实现，不能是抽象方法，静态方法只能访问所在类的静态字段和方法，方法中不能有this和super关键字
  3. 静态语句块：在类初始化时执行一次
  4. 静态内部类：非静态内部类依赖于外部类的实例，也就是说需要先创建外部类实例，才能用这个实例去创建非静态内部类。而静态内部类不需要。
  5. 静态导包：在使用静态变量和方法时不用再指明 ClassName，从而简化代码，但可读性大大降低。

初始化时，静态变量和静态语句块优先于实例变量和普通语句块，最后才是构造函数的初始化

存在继承的情况下，初始化顺序为：

父类（静态变量、静态语句块）

子类（静态变量、静态语句块）

父类（实例变量、普通语句块）

父类（构造函数）

子类（实例变量、普通语句块）

子类（构造函数）

在静态方法中调用非静态成员为什么是非法的

这个需要结合 JVM 的相关知识，静态方法是属于类的，在类加载的时候就会分配内存，可以通过类名直接访问。而非静态成员属于实例对象，只有在对象实例化之后才存在，然后通过类的实例对象去访问。在类的非静态成员不存在的时候静态成员就已经存在了，此时调用在内存中还不存在的非静态成员，属于非法操作。

Object通用方法

概览

euqals

满足5个条件：自反性、对称性、传递性、一致性、与null的比较

  • 自反性
  • 对称性
  • 传递性
  • 一致性(多次调用equals的结果不变)
  • 与null的比较，任何不会null的对象调用equals(null)的结果都为false

equals与==

  • 对于基本类型，== 判断两个值是否相等，基本类型没有 equals() 方法。
  • 对于引用类型，== 判断两个变量是否引用同一个对象，而 equals() 判断引用的对象是否等价。

实现

判断是否是同一个对象的引用，如果是直接返回true；

判断是否是同一个类型，如果不是直接返回false；

将object对象进行转型，判断每个关键域是否相等。

hashCode

hashCode() 的作用是获取哈希码，也称为散列码；它实际上是返回一个 int 整数。这个哈希码的作用是确定该对象在哈希表中的索引位置。hashCode()定义在 JDK 的 Object 类中，这就意味着 Java 中的任何类都包含有 hashCode() 函数。另外需要注意的是： Object 的 hashcode 方法是本地方法，也就是用 c 语言或 c++ 实现的，该方法通常用来将对象的 内存地址 转换为整数之后返回。

等价的两个对象的哈希值一定相同，哈希值相同的两个对象不一定等价

toString

clone

一个类必须要显式重写clone()方法并实现Cloneable的接口，其他类才能直接调用该类实例的clone()方法

浅拷贝

对于基础数据类型采用值传递，对于引用类型，拷贝对象和原始对象的引用类型引用同一个对象

深拷贝

对于基础数据类型采用值传递，对于引用类型，拷贝对象和原始对象的引用类型引用不同对象，即创建一个新对象，并复制其内容

clone的替代方案：拷贝构造函数或拷贝工厂

继承

访问权限

  1. 子类拥有父类对象所有的属性和方法（包括私有属性和私有方法），但是父类中的私有属性和方法子类是无法访问，只是拥有。
  2. 子类可以拥有自己属性和方法，即子类可以对父类进行扩展。
  3. 子类可以用自己的方式实现父类的方法。

抽象类和接口

抽象类不能被实例化，只能被继承，包含抽象方法的类只能是抽象类；接口是抽象类的延伸，Java8之后，接口也支持了默认的方法实现（维护成本太高），接口成员的访问权限只能是 public，字段只能是static和final的

比较：

抽象类提供一种IS-A关系，需要满足里式替换原则，接口提供一种LIKE-A关系（组合）一个类可以实现多个接口，但不能继承多个抽象类

接口的字段都是static和final类型的，成员访问权限只能是public，抽象类没有这些限制

super

子类可以用super()函数访问父类的构造函数，另外，子类可以通过super关键字引用父类的方法实现

重写和重载

重写：子类实现了一个与父类在方法声明上完全相同（方法名和参数列表相同）的一个方法子类重写父类方法，需要满足三个限制条件：子类方法的访问权限必须大于等于父类方法；子类方法的返回类型必须是父类方法返回类型或为其子类型；子类方法抛出的异常类型必须是父类抛出异常类型或为其子类型。

重载：存在于同一个类中，指一个方法与现有方法名称相同，但是参数类型、个数、顺序至少有一个不同。注意：返回值不同，其他都相同不算是重载

反射

Java特征之一，允许运行中的Java程序获取自身信息，并且可以操作类或对象的所有属性和方法。

主要功能（运行时）

获得Class对象判断一个对象是否是某个类的实例（isInstance方法）

构造任意一个类的对象（newInstance方法）

判断任意一个类所包含的成员变量和方法

调用任意一个对象的方法（Invoke方法）

主要应用

开发各种通用配置化框架，在程序运行时根据配置文件加载不同对象或类，调用不同方法

注意事项

反射会额外消耗一定的系统资源，如果不需要动态创建一个对象，则不需要用反射。由于反射调用方法时无视权限检查，因此可能会破坏封装性而导致安全问题，并且由于泛型参数的安全检查发生在编译时，反射可以无视泛型参数的安全检查。

异常

Error

JVM无法处理的错误

Exception

  1. 受检异常，用try…catch语句捕获并进行处理，可以从异常中恢复，常见的受检查异常有： IO 相关的异常、ClassNotFoundException 、SQLException等。
  2. 非受检异常，程序运行时错误，会导致程序崩溃且无法恢复。RuntimeException 及其子类都统称为非受检查异常，例如：NullPointerException、NumberFormatException（字符串转换为数字）、ArrayIndexOutOfBoundsException（数组越界）、ClassCastException（类型转换错误）、ArithmeticException（算术错误）等。
